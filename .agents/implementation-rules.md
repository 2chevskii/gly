# Правила реализации

## Общие правила

- Реализовывать маленькими вертикальными срезами: import smoke test, затем registry, затем matching, затем rendering, затем public commands.
- Сохранять публичный API из `docs/spec/full-spec.md`, если задача явно не меняет спецификацию.
- Не добавлять зависимости без необходимости.
- Не добавлять persistent config до отдельного решения.
- Не расширять MVP возможностями из future work.

## PowerShell стиль

- Целевая версия: PowerShell `7+`.
- Публичные команды используют noun `Gly`.
- Имена файлов публичных команд совпадают с именами команд.
- Private helpers получают префикс `Gly` или явно относятся к `Gly`, чтобы избежать коллизий при dot-source.
- Ошибки пользовательского API должны быть понятными и actionable.
- Formatter helpers не должны выбрасывать необработанные исключения во время вывода объектов.

## Format data

- Загружать `*.format.ps1xml` через `Update-FormatData -PrependPath`.
- Помнить, что format data живет на уровне сессии, а не внутри module scope.
- Не рассчитывать на доступность private-функций из format view без проверки.
- Если нужен exported helper для format view, держать его минимальным и документировать как implementation detail.
- Standard table view должен менять только колонку `Name`.

## Темы и glyph sets

- Built-in entries считать immutable.
- Пользовательские entries регистрируются только из `hashtable` / `pscustomobject`.
- Приоритет правил: позднее правило побеждает раннее.
- Если `Default` не задан и правило не найдено, стиль/glyph не применяется.
- Не делать автоматический fallback с `NerdFonts` на другие наборы.

## Производительность

- Не читать содержимое файлов.
- Не делать Git-запросы.
- Не делать дополнительных filesystem-запросов в горячем пути formatter-а без сильной причины.
- Предпочитать заранее подготовленные lookup-таблицы для имен и расширений.

## Документация

- Пользовательские документы писать на русском.
- Для ограничений PowerShell format data писать прямо: после `Remove-Module` view может остаться активным до конца сессии.
- Для Nerd Fonts писать прямо: модуль не может надежно определить активный шрифт терминала.
